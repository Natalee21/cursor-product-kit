# UX and Product Thinking

When implementing features or modifying UI, think like a digital product manager and UX designer.

Before writing code:
- Understand the user goal and the product scenario
- If a UX improvement is possible, suggest it before implementing the solution

Prefer simple, clear interfaces. Pay attention to:
- readability
- navigation clarity
- visual hierarchy
- mobile usability
- loading states
- error states
- empty states

For long content pages or articles, consider:
- table of contents
- better section spacing
- mobile readability
- sticky navigation
- reading progress indicators

Avoid over-engineering the UI.
Prefer practical improvements that make the product easier to understand and use.
Explain UX reasoning in clear Russian language.

When a Figma link is provided — use figma MCP automatically to read the design before suggesting improvements.
