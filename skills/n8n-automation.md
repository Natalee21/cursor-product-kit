# N8N Automation

## Назначение
Проектировать и создавать автоматизации в n8n — от простых триггеров до сложных мультишаговых воркфлоу.
Применять когда пользователь хочет убрать ручные действия, связать несколько сервисов или создать контент-пайплайн.

## Мышление при проектировании воркфлоу

Перед тем как предлагать схему, нужно понять три вещи:
1. **Триггер** — что запускает процесс? (новое сообщение, расписание, webhook, ручной запуск)
2. **Данные** — что обрабатывается? (текст, файл, данные из формы, API-ответ)
3. **Действие** — что должно произойти на выходе? (отправить, сохранить, уведомить, сгенерировать)

## Типичные воркфлоу для Natali

### Контент-пайплайн для Telegram-канала
```
Триггер: расписание (Cron) каждый день в 9:00
→ OpenAI: сгенерировать идею поста по теме дня
→ Telegram: отправить в личку на согласование
→ Ждать ответа (Wait node)
→ Telegram: опубликовать в канал при получении "ок"
```

### Сбор лидов с лендинга в Notion
```
Триггер: Webhook (форма на сайте)
→ Notion: создать строку в базе данных
→ Gmail: отправить welcome-письмо новому лиду
→ Telegram: уведомить себя о новом лиде
```

### AI-обработка голосовых заметок
```
Триггер: новое голосовое сообщение в Telegram
→ Telegram: скачать аудиофайл
→ OpenAI Whisper: транскрибировать
→ OpenAI GPT: структурировать в задачу или заметку
→ Notion: добавить в inbox
```

### Мониторинг упоминаний и конкурентов
```
Триггер: RSS-фид или HTTP Request каждые 2 часа
→ Проверить новые записи
→ IF: содержит ключевые слова
→ Telegram: отправить дайджест
```

## Узлы n8n, которые используются чаще всего

| Узел | Для чего |
|------|---------|
| Webhook | Получать данные от внешних сервисов |
| Cron / Schedule | Запускать по расписанию |
| HTTP Request | Обращаться к любому API |
| OpenAI | Генерировать текст, транскрибировать, анализировать |
| Telegram | Отправлять и получать сообщения |
| Google Sheets | Читать и записывать данные в таблицы |
| Notion | Работать с базами данных |
| Gmail | Отправлять письма |
| IF / Switch | Ветвить логику в зависимости от условий |
| Wait | Делать паузу и ждать следующего триггера |
| Set | Трансформировать данные между узлами |
| Code | Выполнять JS-код для сложной логики |

## Интеграция n8n с SaaS-проектами Natali

**VoiceReels** — автоматическая отправка уведомления пользователю по email/Telegram после завершения обработки видео.
**Blog Monetization Navigator** — сбор email из формы → Notion CRM → welcome-последовательность.
**borisova.one** — контактная форма → Telegram-уведомление + запись в Google Sheets.

## Деплой n8n
Если нужен собственный инстанс — поднимать через Railway или Render (проще чем VPS).
Для тестирования использовать n8n.cloud (бесплатный план есть).
Webhook URL из n8n подставлять в Vercel Environment Variables для связи с Next.js-проектами.

## Безопасность
Никогда не хранить API-ключи прямо в узлах — использовать раздел Credentials в n8n.
Webhook-эндпоинты защищать секретным токеном в заголовке запроса.

## Формат ответа
Сначала нарисовать схему воркфлоу словами (Триггер → Шаг 1 → Шаг 2 → Результат).
Потом объяснить настройку каждого узла по порядку.
Если что-то нельзя сделать в n8n без кода — честно об этом сказать и предложить Code-узел.
