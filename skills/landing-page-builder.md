# Landing Page Builder

## Назначение
Создавать и улучшать лендинги и продуктовые страницы — от структуры и копирайтинга до кода компонентов.
Применять когда нужно спроектировать новую страницу, переработать существующую или написать убедительный текст для конкретного блока.

## MCP-инструменты
При наличии ссылки на Figma — автоматически использовать figma MCP для чтения дизайна.
При вопросах по библиотекам — автоматически использовать context7 MCP.

## Стек по умолчанию
Next.js (App Router), TypeScript, Tailwind CSS, Framer Motion для анимаций.

## Анатомия сильного лендинга

### 1. Hero-секция (первый экран)
Самый важный элемент — пользователь принимает решение остаться за 3 секунды.

Обязательные элементы:
- Заголовок: конкретная выгода, не описание продукта («Автодубляж видео с клонированием голоса» вместо «Платформа для работы с медиаконтентом»)
- Подзаголовок: уточняет для кого и как работает
- CTA-кнопка: одна, с глаголом действия («Попробовать бесплатно», «Начать сейчас»)
- Social proof или визуал продукта рядом с текстом

### 2. Проблема / боль
Описать ситуацию, которую узнаёт целевой пользователь.
Не перечислять функции — рассказать о боли, которую они снимают.

### 3. Как это работает (How it works)
3–4 шага максимум.
Простые глаголы действия: «Загрузите», «Выберите», «Скачайте».
Можно добавить скриншоты или иллюстрации процесса.

### 4. Преимущества / фичи
Группировать по выгоде для пользователя, а не по технической логике.
Иконка + заголовок + 1–2 предложения объяснения.
Не больше 6–8 блоков.

### 5. Social proof
Отзывы с именем, фото и конкретным результатом — не абстрактные «отличный продукт».
Цифры и кейсы работают лучше звёздочек.

### 6. Pricing (если есть)
Максимум 3 тарифа. Средний тарифный план визуально выделить.
Показывать что входит в каждый, а не только цену.
CTA на каждой карточке.

### 7. FAQ
5–7 вопросов, которые реально задают перед покупкой.
Отвечать честно — не рекламно.

### 8. Финальный CTA
Повторить главный призыв к действию с другим углом мотивации.
Можно добавить гарантию или напоминание о бесплатном входе.

## Копирайтинг для лендингов

### Принципы
Писать для одного конкретного человека, не для «всех».
Использовать язык пользователя, а не профессиональный жаргон.
Конкретика вместо прилагательных: не «быстрый», а «дубляж за 2 минуты».
Один блок — одна мысль.

### Формулы заголовков
- [Результат] без [боли]: «Озвучьте видео на русском без найма диктора»
- [Аудитория] + [результат]: «Для блогеров, которые хотят выйти на зарубежную аудиторию»
- Вопрос-боль: «Теряете просмотры из-за языкового барьера?»

## Технические компоненты (Next.js / Tailwind)

### Hero-секция
```tsx
export function Hero() {
  return (
    <section className="min-h-screen flex items-center justify-center px-6">
      <div className="max-w-4xl mx-auto text-center">
        <h1 className="text-5xl font-bold tracking-tight mb-6">
          Заголовок с выгодой
        </h1>
        <p className="text-xl text-neutral-400 mb-10 max-w-2xl mx-auto">
          Подзаголовок с уточнением для кого и как
        </p>
        <button className="bg-white text-black px-8 py-4 rounded-xl text-lg font-medium hover:bg-neutral-100 transition">
          Попробовать бесплатно
        </button>
      </div>
    </section>
  )
}
```

## Стиль общения
Всегда отвечать на русском языке.
Копирайтинг для лендингов писать на том языке, который указан для продукта.
Давать конкретные примеры текстов, а не общие советы.
